# 페이지 라우팅은 어떻게 되는거지?
리액트는 `react-router-dom`이 담당한다. 이를 사용하지 않으면 a, href를 사용해야 하는데 이는 새로고침을 유발하기 때문에 상태관리가 불가능하다. 즉 SPA

<br/> 넥스트는 next.js의 서버가 페이지를 렌더링 하고 응답한다(SSR/SSG)

# API 라우팅은?
둘다 백엔드 도메인으로 보낸다.

단, 만약 백엔드 도메인이 프론트와 동일하다면 next.js의 서버를 거친다.

# nginx, apache?
웹서버이다. 쉽게말해서 중개서버. 리버스프록시(클라이언트 요청을 서버로 배달), 로드밸런싱 역할을 한다.

프론트에서 요청을 보내면 웹서버가 이를 받아 백엔드로 보내준다고 생각하면 된다. 이게 없다면 요청을 받을 수 없다..? 좀더 알아봐야겠음..
