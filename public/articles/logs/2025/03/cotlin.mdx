# primitive type

코틀린은 객체지향언어라 모든 걸 객체로 만들지만 예외가 있다. 객체를 만들면 heap영역에 공간을 잡고 주소를 반환하는데, 숫자 등은 이렇게 사용하는 게 공간낭비가 될 수 있다.
`primitive type`, wrapper class가 없게 만들어 바로 직접적으로 저장되어있는 공간을 사용한다.

JAVA같은 경우는 `primitive type`, `objective type`이 있지만 코틀린은 그렇지 않다. 사용힐때 객체처럼 사용해도 되고, 리터럴하게 사용할 수도 있다.

코틀린은 auto conversion이 불가능하다. 그냥 숫자연산에 대해 여러 타입의 파라미터를 받을 수 있도록 오버로딩 되어있다.

Arrays과 List는 다르다.

mutable하다는 것은 변수값이 rewritable하다.
