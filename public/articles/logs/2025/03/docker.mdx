# docker
도커가 궁금해졌다. AWS가 서버를 배포하기 위해 존재하는데, 도커는 왜 있는거지?
도커가 밀키트처럼 패키지를 관리하는 거고, 이미지형태로 이를 사용할 수 있다는 것은 알고있지만 실제로 어떻게 동작하는 지가 궁금했다.

<br/> 백엔드 서버를 돌리기 위해서 원격서버가 필요하고 DB가 필요하다. 이러한 서버를 제공해주는 것이 AWS와 같은 클라우드 서비스이다.
하지만 서버가 열려있다고 바로 사용할 수 있는 것은 아니다. 서버를 사용하기 위한 환경을 설정해주어야 하는데, 모든 컴퓨터에서 동일한 환경을 유지하기 어렵기 때문에 Docker를 사용해 컨테이너를 만든다.
컨테이너는 일종의 패키지이기 때문에 어디서 실행하든 동일한 환경을 보장한다. 
컨테이너 안에 필요한 실행 환경을 설정해두고 컨테이너를 실행하면, AWS의 서버에서도, 로컬에서도, 혹은 다른 클라우드 환경에서도 동일하게 실행할 수 있다.
AWS에 컨테이너를 올려야 서버를 돌릴 수 있는 것은 아니지만, 유지보수와 자동화가 훨씬 간편하다.
