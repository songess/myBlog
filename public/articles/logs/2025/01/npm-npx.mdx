# yarn

나는 yarn을 사용해본 적이 없다. 이전에는 속도차이가 있었다고 하지만, 현재는 그다지 차이가 없다고 한다.

또한 `yarn.lock`파일을 통해 로컬에 있는 모든 파일이 같은 파일을 다운받을 수 있게 보장함으로써 버전에 따른 에러가 거의 안뜬다고 하지만, 이것과 관련하여 `npm`을 사용할 때도 큰 어려움을 겪은 적이 없던 터라 사용해야할 이유를 찾지 못했다.

# npm

`node package manager`이다. 말 그대로 package를 다운받고 이를 관리한다. 이 과정에서 `package.json`에 의존성이 추가된다.

# npx

`npm package execution`이다. 패키지를 설치하는 것이 아닌 실행만한다. 임시로 다운받아 실행하는 것이기 때문에 `package.json`에 의존성이 추가되지 않는다.
단, 스크립트를 실행하는 과정에서 내부적으로 install이 실행된다면 의존성이 추가될 수 있다.

동작방식은 최신버전의 파일을 임시로 불러와 실행시키고 삭제한다(일회용). 원하는 코드만 복붙해온다고 생각할 수 있다.

- 이는 특정 노드 버전에 스크립트 실행이 필요할 때 사용할 수 있다.
- npm은 의존성을 관리해야 하는데 이 부분이 불필요하다 느낀다면 npx를 사용할 수 있다.
- npm을 통해 설치할 때 필요없는 파일이 같이 불러와진다고 생각되면 필요한 파일만 npx를 사용해 설치할 수 있다.
